# Regenerate API Key

Generate a new API key for your OpenFiles project. This will invalidate the previous API key immediately. This endpoint requires JWT authentication.

<Warning>
  **This action cannot be undone.** Your old API key will stop working immediately after regeneration.
</Warning>

## Authentication Required

This endpoint requires JWT authentication (not API key). Use your GitHub-authenticated session token.

## Use Cases

- **Security incident response** - Rotate keys after potential compromise
- **Key rotation policy** - Regular security maintenance
- **Team changes** - Replace keys when team members change
- **Lost key recovery** - Generate new key when previous is lost

## Example Usage

<CodeGroup>
```javascript JavaScript
const response = await fetch('https://api.openfiles.com/project/apikey/regenerate', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${JWT_TOKEN}`,
    'Content-Type': 'application/json'
  }
})

const result = await response.json()
console.log('New API key:', result.api_key)
// Save this key securely - it won't be shown again
```

```python Python
import requests

response = requests.post(
    'https://api.openfiles.com/project/apikey/regenerate',
    headers={'Authorization': f'Bearer {JWT_TOKEN}'}
)

result = response.json()
print('New API key:', result['api_key'])
# Save this key securely - it won't be shown again
```

```bash cURL
curl -X POST 'https://api.openfiles.com/project/apikey/regenerate' \
  -H 'Authorization: Bearer YOUR_JWT_TOKEN'
```
</CodeGroup>

## Response

Returns the new API key. **Store this securely** - it will not be displayed again.

```json
{
  "api_key": "oa_new1234567890123456789012345678901234567890",
  "created_at": "2024-01-15T10:30:00.000Z",
  "expires_at": null
}
```

## Security Notes

- The old API key is invalidated immediately
- Update all applications using the old key
- Store the new key in a secure location (environment variables, secrets manager)
- Never commit API keys to version control
